DAY 5 NOTES
============

Object-Oriented Programming
---------------------------

1. Object-oriented programming organizes code using classes and objects.
2. An object is an instance of a class.
3. When instance variables are used inside methods, those methods are called instance methods.

Static Method
-------------

A static method is used when the code belongs to the class but does not depend on object state.

Key points:
- it does not use instance variables
- it does not use class variables by default
- `self` and `cls` are not required
- it is declared using the `@staticmethod` decorator

Inheritance
-----------

Inheritance means extending properties from one class to another class.

- Base class or parent class:
  the class that gives its properties to another class
- Derived class or child class:
  the class that inherits those properties

Types of inheritance:
1. Single inheritance
2. Multilevel inheritance
3. Multiple inheritance

Polymorphism
------------

Polymorphism means having many forms. It is the ability of the same message or method name to behave differently in different situations.

Overloading
-----------

Python supports operator overloading, but it does not support traditional method overloading or constructor overloading like some other languages.

Function overloading means using the same function name with different arguments, but Python usually handles this with default arguments or flexible parameters.

Overriding
----------

Python supports method overriding and constructor overriding.
